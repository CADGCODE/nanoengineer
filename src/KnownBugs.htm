This file documents some bug fixes, unsupported features and some of the commonly encountered bugs in nanoENGINEER-1 Alpha7.

UNSUPPORTED FEATURES:

 

#UNDO/REDO

- Undo/Redo knows nothing about the mode the program was in. For example, if you extrude a moiety while in extrude, return to your default mode and then hit Undo, it will only separate the base and repeat units. It will not go back to extrude mode. (1391)

- MMKit tab and button changes and most changes made in the dashboards are not recorded in the Undo stack

- On large models, undo and other operation might become slower. Further performance improvements will be implemented in a future release of the software

 

#PDBs With Multiple Bond Information Not Supported

 This version of ne-1 treats all bond types as single bonds while reading from or writing to a PDB file. Also, it doesn't save the hybridization type into the PDB file. For example, (sp2) hybridized Carbon in a structure will be saved as a sp3 hybridized Carbon with wrong bond valence. (3 bonds instead of 4). Also, the double bond of this (sp2) Carbon is saved as a single bond Therefore, save your models ONLY in MMP file format to avoid major loss of work (569, 763)

 

SIMULATOR / MINIMIZE

# Simulator/Minimize: will support bonding between elements specified in the element selector of nanoENGINEER-1.

# Simulator/Minimize: Multiple bonding between certain elements is generally supported, but some parameters, notably for double bonds involving nitrogen, are estimated.  

# Simulator/Minimize: The simulator doesn't take into account the effect of torsion. This will make systems with double or aromatic/graphitic bonds act less stiff than they really would.   

# Simulator: Electrostatic interactions are not supported; in particular, the hydrogen and dative bonds are not handled.

# Enable motor in minimize feature is experimental. Enabling a motor in minimization may result in high values of rms force. Also, minimize will take long time to finish

# Nanotubes: The simulator doesn't take into account the effect of torsion. The simulation/minimization of nanotubes and similar structures is not supported.

 

GAMESS Jig

#GAMESS jig can be added by select some atoms first in 'Select Atoms' mode and then attaching this jig by using Menu item: Jigs>Gamess. This jig can be attached to a maximum of 200 atoms.

-- Certain GAMESS optimization runs or Energy calculation can take very long time (hours/days). While a GAMESS job is being performed, user is not allowed to do any other operation in that session of ne-1. User must wait till the GAMESS job is finished or opt to terminate it by hitting 'Cancel' button in the progress bar.

-- Also, certain GAMESS run may create large output files. So make sure that your system has enough disk space to handle such runs. ALL GAMESS jobs are save inside the 'Job Manager' directory. (User's home directory/ Nanorex/Job Manager) (742)

 

ESP IMAGE / NANO-HIVE

# ESP Image jig is not fully supported on Linux and MAC platforms. On Windows, you can perform ESP calculations for some standard cases provided that a correct version of NanoHive is installed. In general ESP image is partially supported and has many known bugs. These issues will be fixed in an upcoming release of nanoENGINEER-1.

 

# Bond Inference not supported but a limited valence error check supported.

 

By the term 'bond inference' we mean automatically deciding what type of bond will be formed between the participating elements. It is not supported for this version of ne-1. When an element is deposited, by default, all the bonds are single bonds. User must manually change the bond type between the elements in 'Build mode'. However, if the structure is not chemically correct (bad valences etc), ne-1 shows a 'pink wire frame' around each atom indicating valence errors.  This 'valence error' display, by default, is enabled. It can be changed from the 'Bonds' tab in the Preferences dialog. (Edit>Preference…).  

Note that there are some kinds of chemical errors this (the Valence Error Checker) doesn't detect. For example, it can't tell whether rings should be aromatic or not. All it checks is that the bonds on each atom could possibly add up to the right number of valence electrons. The Valence Error Checker doesn't treat graphitic and aromatic bonds differently.

MEMORY AND PERFORMANCE PROBLEMS

# Memory consumed by the application increases due to repeated execution of certain operations such as opening and closing a file (often termed as ‘memory leaks’. This affects the overall performance of the software and could result in a crash. This version doesn’t prevent such memory leaks.

 

# nE-1 has some known performance problems while using some features. As a result, some operations take considerably longer times. This is particularly true when you are handling a large file.

Examples: Opening a large file, Operations performed in Fuse mode, moving selected atoms, Selecting and deselecting some atoms from a large chunk, deleting some atoms, Undoing or redoing these operations, playing a movie on a large model etc. may take longer time when the file size is large. (991, 1473, 1674, 1704, 1750, 1778)
