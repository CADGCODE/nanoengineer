nanoENGINEER-1™ v0.0.4 (Alpha 4)

 

 

Bug Fixes In Alpha 4

 

  • Bug #130: ‘Hydrogenate’ add hydrogen atoms to all open bonds above waterline in Build mode. Thus, its implementation is different than Dehydrogenate and Passivate in that mode (these two work  only on a selection). Now, Hydrogenate is implemented in such a way that it works only when a chunk is selected.

 

  • Bug #360 (item 8): Single cut operation for multiple selected objects places copies of only alternate objects in the clipboard.

 

  • Bug #381: Deleting a group (by selecting it in the Model Tree and pressing the Delete key) appears to delete the selected chunks, but they can still be available in Extrude mode.

 

  • Bug #386 (item 2): File->Close operation after minimizing an opened model closes the file without saving changes due to minimize(and without prompting the user to save the changes made after minimization)

 

  • About nanoENGINEER-1 dialog background now displays the proper metallic/aqua color on MacOSX.

 

 

Unsupported Features:

 

  1. Undo/Redo not implemented.

 

  1. Cannot change color of individual atoms.(189)

 

  1. Can not move, translate or rotate a chunk by editing its coordinates or providing an x, y, z offset. (9)

 

  1. Points, lines, and planes are not implemented yet. (47)

 

  1. Simulator will support bonding between elements specified in the element selector of nanoENGINEER-1. Multiple bonding is not supported for alpha.

 

  1. The 'Clipboard' has limitations and some bugs.

 

  1. Do not drag and drop anything between the model tree and the clipboard.

 

  1. The symmetry information for a selected chunk cannot be obtained. (261)

 

  1. sp and sp2 bonds are not supported.

 

  1. Aromatic bonds are not supported.

 

  1. Bonds are not selectable.

 

  1. Linear motors are not supported by the simulator. (393)

 

 

Known Bugs (All Platforms)

 

  1. Delete operation cannot be undone. In fact, Undo/Redo are not implemented yet. SAVE YOUR WORK OFTEN!

 

  1. If you select the part icon (top icon in the model tree) and hit the delete key (or click Delete), the entire part will be deleted without asking the user for confirmation. (147)

 

  1. During Minimize, the progress bar will stay at 0% until the last moment, then jumps to 100% before hiding.

 

  1. Only open one main window. Opening more than one nanoENGINEER-1 application window will have unexpected results. (362)

 

  1. Dragging and dropping objects onto the part icon at the top of the list places the items(s) at the bottom of the list.

 

  1. Dragging and dropping items onto a group folder places them at the bottom of the group, not the top.

 

  1. Dragging and dropping items between the clipboard and model tree can sometimes have unexpected results. Use cut, copy and paste instead.

 

  1. The front clipping plane will sometimes clip objects that should not be clipped.

 

  1. Align to common axis averages the axis vectors of the selected chunks. The resultant vector is (usually) different than the axis vectors of any of the chunks. (187)

 

  1. Any file currently opened by nanoENGINEER-1 can be manually deleted from the computer without any warning message. (215)

 

  1. After drawing a 'cookie' in the cookie cutter, if the orientation view is changed and then any new cookie drawn will be placed at a wrong location. Thus, the user should not change the views while in the cookie cutter. (236)

 

  1. User can not save exclusive features of nanoENGINEER-1 into the PDB file format. nanoENGINEER-1 doesn't give any warning message for users trying to save such features in a .pdb file. (252)

 

  1. There is a display bug associated with a cut operation on a chunk. It stretches the inter-chunk bond and keeps bonded portion of cut chunk visible. If this happens, select the visible chunk and use "Delete Bonds" to remove the bonds between the two chunks. (371)

 

  1. Extrude Tool: The 'extrude a ring' feature has some bugs. These will be fixed for the Beta release of nanoENGINEER-1. To use extrude's 'Ring Mode' effectively, make sure you follow these steps:

 

-- Get everything adjusted first in the 'rod mode'. For example, adjust the number of copies (N), check boxes on the dashboard about the final product and the way you want bonding between chunks to form.

 

-- Then you may switch to "Rod Mode" by selecting ring from the rod pulldown menu on the dashboard. While viewing the ring, you may increase the number of copies.

 

-- If you need to do any further adjustments, switch back to "Rod Mode", make changes and switch back to the "Ring Mode".

 

-- Select the 'Done' button on the dashboard when finished. (155)

 

  1. Extrude Tool: Multple chunks are automatically merged into a single chuck, even when cancel is hit. (397)

 

  1. When starting a new part or opening a new file, the object name indexing is not reset to 1. For example, new chunk names continue to increment from the previous part's index number. (64)

 

  1. The MMP File Format specification included in the nanoENGINEER-1 Assistant is obsolete. The most current version can be found at www.nanorex.com/mmpformat.htm

 

  1. Rotary and Linear motors do not move with the chunks they are attached to. (168)

 

  1. Cannot select a jig (i.e. motor, ground, thermostat or thermometer) in the graphics area. Jigs may only be selected via the model tree. (6)

 

  1. "Select Doubly" can often take a long time to complete. (102)

 

  1. Middle mouse button "Hint Cursors" can sometimes get stuck on. Pressing the Shift or Cntl/Cmd key will usually fix the cursor. (113, 158)

 

  1. Build Mode: The skull and crossbones cursor will not display (when pressing Ctrl/Cmd) after clicking in the Model Tree area. (217)

 

  1. Cookie Mode: Changing views (Front, Top, Right, etc.) and then making a selection does not select properly. (236, 277)

 

  1. Cookie Mode: The thickness of the current layer does not change once selection has started, even though the thickness can be changed on the dashboard. (167)

 

  1. Cookie Mode: When adding a new layer, there is no visual cue indicating a new layer has started. (281)

 

  1. Cookie Mode: Large selections can cause response latency. (8)

 

  1. Display: Larger atoms can obscure open bonds when displayed in VdW mode. (258)

 

  1. Movie Player: If you immediately select the Movie Player after Minimize, you will get the error message "Movie file [/homedir/Nanorex/sim-PID#.xyz] not valid for the current part".

 

  1. Simulator: The parameters in the simulator always reset to 900 frames, 10 steps per frame and 300K rather than the last parameters entered from the previous simulation run.

 

  1. Zoom Tool: It is still experimental. Some Mac users report cosmetic bugs. It is a good idea to save your work before using it.

 

  1. There are some bugs that occur when a blank PDB file is inserted into any open file using File->Insert. Therefore, avoid inserting a blank PDB file. (433)

 

  1. Hydrogenate/Dehydrogenate/Passivate work for clipboard objects (411)

 

  1. Simulator: in some cases, open bonds ‘fly away’ from the elements while running a simulation movie. (254)

 

  1. Minimize: Large values written to the MMP (or to the generated XYZ) file after minimizing a model containing overlapped open bonds or excessively stretched bonds (422)

 

 

  1. Some bugs occur if one tries to play an invalid movie file for a part. This may result in loss of data. Therefore, do not try to play a movie file that is invalid for the opened part. (384)

 

  1. Can not select a chunk if cursor is inside the selection polyhedron but not pointing inside perimeter of any of the atoms in that chunk. (11)

 

  1.  In the Cookie Cutter, Shift+Left Mouse drag only deletes drawing that is outside but doesn't draw inside the lasso/ rectangle (190)

 

  1. Extrude: Separates the chunk permanently when atoms are partially selected and then extrude mode is cancelled. (198)
