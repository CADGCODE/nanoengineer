Unsupported Features:

 

  1. Undo/Redo not implemented.

 

  1. Bond Inference not supported .

 

By the term ‘bond inference’ we mean automatically deciding what type of bond will be formed between the participating elements. It is not supported for this version of nE-1. When an element is deposited, by default, all the bonds are single bonds. User must manually change the bond type between the elements in ‘Build mode’. However, if the structure is not chemically correct (bad valences etc), nE-1 shows a ‘pink wire frame’ around each atom indicating valence errors.  This ‘valence error’ display, by default, is set to ON. It can be turned On or Off from the ‘Bonds’ tab in the Preferences dialog. (Edit>Preference…). But also note that there are some kinds of chemical errors this doesn't detect. For example, it can't tell whether rings should be aromatic or not. All it checks is that the bonds on each atom could possibly add up to the right number of valence electrons.

 

Example:

-- In Build mode, select ‘Carbon’ and then change its hybridization to ‘sp’ by clicking on the appropriate button in this dialog.

-- Deposit this (sp) Carbon in the 3D Workspace. Observe that both open bonds of the deposited are single bonds and a pink wire frame appears around the atom center indicating valence errors.

-- Now select (sp) Nitrogen from the Modeling Kit

-- Highlight one of the open bonds of (sp) carbon and deposit this (sp) Nitrogen, Notice that a single bond will be formed between the C(sp) and N(sp). But nE-1 will display valence error wire frames around these atoms.

-- You can change this bond type to a triple bond by:  (a) highlight the open bond, right click and then from the context menu that popped up, select triple bond. OR (b) from the Build dashboard, click on the button that corresponds to a triple bond, now move your mouse such that the bond gets highlighted and then left click.

-- By following this procedure, you can change the bond order. Observe that, when the triple bond is formed, the valence error wire frames will go away.

 

  1. nanoENGINEER-1 (nE-1) treats all bond types as single bonds while reading from or writing to a PDB file. Also, it doesn’t save the hybridization type into the PDB file. For example, (sp2) hybridized Carbon in a structure will be saved as a sp3 hybridized Carbon with wrong bond valence. (3 bonds instead of 4). Also, the double bond of this (sp2) Carbon is saved as a single bond.  Therefore, save your models ONLY in MMP  file format in such cases to avoid major loss of work (569, 763)

 

Simulator / Minimize

 

  1. Simulator/Minimize: will support bonding between elements specified in the element selector of nanoENGINEER-1. But note that it doesn’t handle bond stretching with Fluorine atom or bond bending when the involved atom is Boron.

 

  1. Simulator/Minimize: Multiple bonding between certain elements is supported but the others are not.  For example structures involving a double bond to nitrogen it is not supported.

 

  1. Simulator/Minimize: The simulator doesn’t take into account the effect of torsion. This will make systems with double or aromatic/graphitic bonds act less stiff than they really would. 

 

  1. Simulator: Electrostatic interactions are not supported. Also, the hydrogen or dative bonds are not handled.

GAMESS Jig

  1. GAMESS jig can be added by select some atoms first in ‘Select Atoms’ mode and then attaching this jig by using Menu item: Jigs>Gamess. This jig can be attached to a maximum of 200 atoms.

-- Certain GAMESS optimization runs or Energy calculation can take very long time (hours/days). While a GAMESS job is being performed, user is not allowed to do any other operation in that session of nE-1. User must wait till the GAMESS job is finished or opt to terminate it by hitting ‘Cancel’ button in the progress bar.

-- Also, certain GAMESS run may create large output files. So make sure that your system has enough disk space to handle such runs. ALL GAMESS jobs are save inside the ‘Job Manager’ directory. (User’s home directory/ Nanorex/Job Manager) (742)

Build Mode

  1. Build: While in this mode, you can not delete an atom in the 3D workspace using Ctrl + left click (Cmd +click on MAC) operation if that atom is represented in lines display. The atom center doesn’t get highlighted while in lines display. For the same reason, you can not move the atom using left mouse drag. Simple workaround exists. Change the display representation to Tubes or CPK or VDW , then highlight the atom center and do Ctrl+ left click or left drag  or shift + left drag operation.(715-2)

 

  1. Build/ Bond Order : The context menu ‘Reposition Open Bonds’ of an atom reorients the open bonds but wile doing so, it makes the open bond type as single  Let’s assume that a Carbon atom with (sp2) hybridization is deposited in build mode. Now you highlight one of its open bonds, right click and select ‘double bond’ from the context menu. The representation of the open bond will change to a double bond representation. Now if you highlight the atom center, right click and select ‘reposition open bonds’ , it will clear the ‘would be a double bond’ representation you just made and thus, all three open bonds of (sp2) carbon will be represented as single bond types. (839)

Extrude Mode

 

  1.  The 'extrude a ring' feature has some bugs. These will be fixed for the Beta release of nanoENGINEER-1. To use extrude's 'Ring Mode' effectively, make sure you follow these steps:

 

-- Get everything adjusted first in the 'rod mode'.  For example, select the proper bond offset sphere or arrange the copy units such that you can see their open bonds highlighted in a way you would like to bond them. (In general, while in the ring mode, adjustments made from graphics area don’t work)

 

-- Then, you may switch to "Ring Mode" by selecting ring from the rod pulldown menu on the dashboard. While viewing the ring, you may increase the number of copies.

 

-- If you need to do any further adjustments, switch back to "Rod Mode", make changes and switch back to the "Ring Mode".

 

-- Select the 'Done' button on the dashboard when finished. (155)
